Qu'est-ce qu'un blueprint ?

Un "exosquelette pédagogique" de l'évaluation

Page HTML interactive pour guider les élèves dans leurs révisions basées sur les compétences

  • Structure du contrôle (types, barème, durée)
  • Compétences attendues (programmes officiels) avec priorités
  • Quiz de compréhension du cours
  • Fléchage intelligent → exercices types de la fiche
  • Conseils de révision personnalisés et actionables

Prérequis

Disposer de la séquence pédagogique complète :

  • Sujet d'évaluation (enonce.tex ou PDF)
  • Synthèse d'évaluation (compétences, barème)
  • Cours du chapitre (contenus attendus)
  • Fiche d'exercices (exercices types et solutions)
  • Activités pédagogiques (démonstrations, investigations)

Le skill analyse toute la séquence pour créer un fléchage intelligent vers les exercices types

Évaluation par compétences

Aligner l'évaluation avec les programmes officiels

  • Programmes de l'Éducation Nationale définissent les compétences attendues
  • Grille de compétences : chaque exercice évalue une ou plusieurs compétences
  • Critères de réussite : ce qu'il faut savoir faire pour chaque compétence
  • Niveaux de maîtrise : non acquise → en cours → acquise → dépassée

Le blueprint organise les révisions autour de ces compétences officielles, pas autour des exercices

Règles de confidentialité

Le blueprint ne doit JAMAIS révéler le contenu exact du contrôle

Autorisé

  • "Exercice 1 : QCM (6 questions)"
  • "Appliquer les formules du produit scalaire"
  • "Problème guidé avec étapes"

Interdit

  • "L'exercice 2 demande de calculer AB · AC"
  • "Le triangle ABC a les sommets A(1,2), B(3,4)"
  • "Démontrer que ABCD est un losange (énoncé exact)"

Bonnes pratiques

À faire

  • HTML autonome (zéro dépendance externe)
  • Design responsive (mobile, tablette, PC)
  • Tenir sur 1-2 pages A4 à l'impression
  • Fléchage précis (exercices numérotés + niveaux)
  • Priorités cohérentes avec le barème
  • Conseils concrets et actionables

À éviter

  • Dépendances externes (CDN, images)
  • Contenu trop dense (> 70% de l'espace)
  • Fléchage vague ("revoir le cours")
  • Toutes les compétences en priorité haute
  • Conseils génériques ou décalés du contrôle
  • Révéler l'énoncé exact du contrôle

Un pattern réutilisable
