Objectif du skill

Créer des notebooks Jupyter pédagogiques structurés selon un protocole quasi-systématique

  • Activités Python pour Seconde, Première, Terminale
  • Progression cognitive : comprendre → appliquer → créer
  • Auto-évaluation intégrée avec QCM
  • Exercices à complétion progressive
Appuyez sur ↓ pour voir les domaines couverts

Thèmes par niveau

Seconde

  • Variables, conditions, boucles
  • Fonctions, listes
  • Aléatoire, graphiques

Première

  • Boucle while, suites
  • Statistiques, probabilités
  • Variations de fonctions

Terminale

  • Méthodes numériques, équations différentielles, Monte-Carlo

Structure pédagogique

Chaque notion suit un protocole en 6 blocs

1
Cours — Explication théorique avec syntaxe
2
Exemple corrigé — Code exécutable commenté
3
QCM vérification — 3 questions de validation

4

# Valider la structure (balises HTML, types)

# Sortie JSON pour traitement

Indices minimalistes

RÈGLE : Les indices sont minimalistes. L'élève doit chercher !

Ce qu'on rappelle

  • Fonction Python vue pour la 1ère fois
  • Format d'affichage imposé
  • Module spécial à importer

Ce qu'on NE rappelle JAMAIS

  • Structures algo (for, if, while)
  • Liste des variables
  • Fonctions déjà vues

Bonnes pratiques

À faire

  • plt.clf() avant matplotlib
  • Commentaires dans le code
  • Cellules ≤ 15-20 lignes
  • Corrections dans <details>
  • QCM avec 3 questions exactement

À éviter

  • Cellules trop longues
  • HTML dans cellules CODE
  • Énoncés vagues
  • Indices trop guidants
  • Corrections qui spoilent

Checklist finale

  • Toutes les cellules de code sont exécutables
  • Les corrections sont dans des <details>
  • Les QCM ont exactement 3 questions
  • La progression est logique
  • Les cellules markdown ne contiennent pas d'émojis dans <summary>
  • L'en-tête est complète (niveau, durée, objectifs)
